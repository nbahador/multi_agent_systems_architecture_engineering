Multi-Agent System Template Guide

Introduction

This interactive textbook teaches you how to use the multi-agent system template to convert your ADK (Agent Development Kit) agent into an A2A (Agent-to-Agent) web application.

What You'll Learn: How to wrap your existing ADK agent in a web server, create HTTP endpoints for agent communication, handle session management, and generate an agent card that describes your agent's capabilities.

What is A2A?

A2A (Agent-to-Agent) is a protocol that allows different AI agents to communicate with each other over HTTP. This enables creating systems where multiple specialized agents work together to solve complex problems.

Why Use A2A?

A2A provides a standardized way for agents to:

  • Discover each other's capabilities
  • Exchange messages and data
  • Coordinate on complex tasks
  • Scale horizontally across multiple servers

Template Structure

The multi-agent system template is organized into four main sections that you need to customize:

# SECTION 1: CUSTOM IMPORTS
# REPLACE THIS SECTION WITH YOUR AGENT-SPECIFIC IMPORTS
from my_agent import MyCustomAgent
from my_tools import CalculatorTool, DatabaseTool

# SECTION 2: CONFIGURATION
# Add any configuration loading here
API_KEY = "your-api-key"

# SECTION 3: AGENT CREATION
def create_my_agent():
    return MyCustomAgent(
        name="My Agent",
        tools=[CalculatorTool(), DatabaseTool()]
    )

# SECTION 4: APPLICATION ENTRY POINT
my_agent = create_my_agent()
app = to_a2a(agent=my_agent)

Template Sections

Section 1: Custom Imports - Replace with your agent-specific imports

Section 2: Configuration - Add environment variables and configuration

Section 3: Agent Creation - Implement your agent creation logic

Section 4: App Initialization - Create your agent and convert it to A2A

Note: The core A2A and ADK imports should not be modified as they provide the essential infrastructure.

Section 1: Custom Imports

In this section, you need to replace the placeholder imports with your actual agent-specific imports.

# ============================================================================
# SECTION 1: CUSTOM IMPORTS
# ============================================================================
# REPLACE THIS SECTION WITH YOUR AGENT-SPECIFIC IMPORTS
#
# Examples of what to add here:
# - Your custom agent class import
# - Any tools or plugins your agent uses
# - Configuration files or environment variables
# - Database connections or external API clients
#
# Example imports:
from my_project.agents.chat_agent import ChatAgent
from my_project.tools.calculator import CalculatorTool
from my_project.config import load_config
import os

Tip: Make sure to import all the dependencies your agent needs to function properly, including any third-party libraries.

Section 2: Configuration

This optional section is where you can set up configuration variables, environment variables, and other settings your agent needs.

# ============================================================================
# SECTION 2: CONFIGURATION (OPTIONAL)
# ============================================================================
# Add any configuration loading or environment setup here
#
# Examples:
CONFIG = load_config()
API_KEY = os.getenv("MY_API_KEY")
DATABASE_URL = os.getenv("DATABASE_URL", "sqlite:///default.db")

Security Note: Never hardcode sensitive information like API keys or database credentials. Always use environment variables or secure configuration files.

Section 3: Agent Creation

This is where you implement the logic to create and configure your custom agent.

# ============================================================================
# SECTION 3: AGENT CREATION AND APP INITIALIZATION
# ============================================================================
# This is where you create your agent instance and convert it to A2A

def create_my_agent() -> BaseAgent:
    """
    Create and configure your custom agent.
    """
    return ChatAgent(
        name="My Chat Agent",
        description="A helpful chat assistant",
        model="gpt-4",
        tools=[CalculatorTool(), WebSearchTool()]
    )

Remember: Your agent must inherit from BaseAgent and implement all required methods. The template provides a placeholder function that you need to replace with your actual implementation.

Section 4: Application Initialization

In this final section, you create your agent instance and convert it to an A2A application.

# ============================================================================
# SECTION 4: APPLICATION ENTRY POINT
# ============================================================================
# This creates the final application that uvicorn will run

# Create your agent instance
my_agent = create_my_agent()

# Convert your agent to an A2A application
app = to_a2a(
    agent=my_agent,
    host="0.0.0.0",
    port=8080,
    public_url=None
)

Running Your Application: Once you've completed all sections, you can run your application with the command: uvicorn your_module:app --host 0.0.0.0 --port 8080

Detailed Flow Explanation
  1. HTTP Request: A client sends an HTTP request to your agent's endpoint
  2. Starlette App: The Starlette web framework receives the request
  3. A2A Handler: The A2A-specific request handler processes the request
  4. Agent Executor: The executor manages the agent's lifecycle and execution
  5. Your Agent: Your custom agent processes the request and generates a response
  6. HTTP Response: The response is sent back through the chain to the client

Best Practices

Error Handling

Always implement proper error handling in your agent:

try:
    # Your agent logic here
    result = agent.process(request)
except Exception as e:
    logging.error(f"Error processing request: {e}")
    return Response("Internal Server Error", status_code=500)
Performance Optimization

For better performance:

  • Use connection pooling for database connections
  • Implement caching for frequently accessed data
  • Consider using asynchronous operations for I/O-bound tasks
  • Monitor your agent's performance with appropriate metrics
Security Considerations

Keep these security best practices in mind:

  • Validate all incoming requests
  • Implement authentication and authorization
  • Use HTTPS in production
  • Sanitize inputs to prevent injection attacks
  • Regularly update dependencies to patch vulnerabilities
